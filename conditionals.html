Conditionals

What are Conditionals for?

Conditionals allow you to have code that only gets executed when a certain condition is satisfied. This means that you can make your code act differently under different conditions.

As an example, conditionals allow me to create a program that tells me "good morning" if the time is before 12:00 p.m. but tells me "good afternoon" if the time is after 12:00 p.m.

Conditionals are able to do this thanks to their structure and the way they are executed.

Structure

The blocks in a Conditional have their identifying text be a condition, and their code be the code to be executed if the condition is true. Conditions are simply python expressions. Because any python expression evaluates to an object, and because all objects evaluate to either True or False, any expression works as a condition. Once you have your blocks in mind, the Conditional is then written by placing the blocks one after another.

How Conditionals are Executed

When your code is run, python will look through each block in order and check its condition. Once it finds a condition that is true, it will execute that block's code and skip the rest of the blocks. So the first component who's condition is true is the only one that has it's code run. This is so that if you have many pieces of code for completely different situations only one will execute per situation. On the other hand, if you want many pieces to execute per situation you can always write several conditionals.

Writing Conditionals

So what are the rules for writing conditionals? As was stated above, writing conditionals is a simple as writing out its blocks one after the other. So here are the rules for those blocks.

Rules for Writing the Blocks that Comprise Conditionals

Keywords

  • First block's keyword is an if
  • Subsequent blocks use elif as their keyword

Identifying Text

Acts as the condition for the code that follows. Any expression is legal. Use the expression that gives you the condition you want.

Code

You can use loops, expressions and/or more conditionals. Other than that it is completely up to you!

Optional Final Block

If you have multiple blocks, there is an optional final block that you can use which does not have identfying text. It looks like this:

else:
 some code

If this block is part of your conditional and none of the other blocks are executed, this one will be. Therefore, it is a shorter way of writing this as your last block.

elif True:
 some code
