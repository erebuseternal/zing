Conditionals

Why

Conditionals allow you to have code that only gets executed when a certain condition is satisfied. This means that you can make your code act differently under different conditions.

What

Conditionals are made of blocks written one after another. Python knows these blocks are tied together because of the keywords they use. Each block's identifying text is an expression that will act as a condition. On execution Python will begin looking at the blocks' expressions in order. Once it finds an expression that evaluates to True, it will execute that block's code and then skip the rest of the blocks. Therefore the first component who's expression is true is the only one that has it's code run. This is so that if you have many pieces of code for completely different situations only one will execute per situation. On the other hand, if you want many pieces to execute per situation you can always write several conditionals.

How

So what are the rules for writing conditionals? As was stated above, writing conditionals is as simple as writing out its blocks one after the other. So here are the rules for those blocks.

Rules for Writing the Blocks that Comprise Conditionals

Keywords

  • First block's keyword is an if
  • Subsequent blocks use elif as their keyword

Identifying Text

Just a python expression. Use the expression that gives you the condition you want.

Code

You can use loops, expressions and/or more conditionals. Other than that it is completely up to you!

Optional Final Block

If you have multiple blocks, there is an optional final block that you can use which does not have identfying text. It looks like this:

else:
 some code

If this block is part of your conditional and none of the other blocks are executed, this one will be. Therefore, it is a shorter way of writing this as your last block.

elif True:
 some code
