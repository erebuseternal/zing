Expressions

An expression in python is a line of code that evaluates to an object. These lines are composed of objects and function calls (we will get to functions later) separated by operators. Operators take two objects and produce a third from them.

Here is a simple expression:

1 + 1

This expression has two objects (in this case numbers) separated by the operator of addition. This of course evaluates to the number 2.

Expressions don't have to have operators in them. Here is an example of the simplest expression you can have:

1

This expression evaluates to one of course.

But once again, expression are simply objects and function calls strung together with operators. And all expressions evaluate to an object.

Order of Operations

Now take the expression 1 + 2 * 3. If we evaluate it using normal order of operations it evaluates to 7. If we evaluate it strictly from left to right we get 6. To remove this ambiguity, python has a strict order of operations. More often than not this is not going to be strictly from left to right. So how are they ordered? We will learn to that as we learn the operations that you can use.

But there is one way that you can always control the order of operations. Any part of an expression surrounded by parenthesis, specifically "(" and ")", will be evaluated before the rest of the expression is evaluated. Therefore:

(1 + 2) * 3

becomes

3 * 3

and similarly

1 + (2 * 3)

becomes

1 + 6

None

None is a special python object that is used to represent the lack of an object. So when a line of code should evaluate to nothing, it evaluates to python's version of nothing, None, which is still an object. Note that like any other object, you can assign None any number of other names.

Booleans: True and False

True and False, called booleans, are two special objects. In python, every object not only evaluates to itself, but also evaluates to one of these two. And the rules for what evaluates to what are quite simple. I'll divide them up by the type of object (and don't worry if some of this doesn't make sense, it will become clear as we move along).

Who Evaluates to What

  • Sequences and Dictionaries
    • Ones without elements evaluate to False
    • Ones with elements evaluate to True
  • Numbers
    • 0 evaluates to False
    • Everynumber that isn't 0 evaluates to True
  • Booleans
    • False is False
    • True is True
  • All Other Objects
    • None evaluates to False
    • Everything else evaluates to True (unless otherwise stated)

Finally there are three ways you can combine True and False to get either True or False: and, not, and or. And they are just like their english versions!

Boolean Operators

and For a and b to be True both a and b have to be True.
or For a or b to be True either a, b, or both have to be True.
not not True is False and not False is True.
