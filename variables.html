Variables

A variable in python is simply a name. Names are how we reference all objects in python. Because names are used to tell things apart, each name can be assigned to at most one object. On the other hand just like a person can have multiple names, each object can have multiple names assigned to it.

How to Assign a New Name

To assign a new name to an object we write:

name = object

  1. The name we want to use
  2. An equals sign
  3. One of the object's old names

Assigning a Name: one old name

Suppose I have a snake object currently given the name cyber_joe. Let's give him a longer name: cyber_joseph. We do so by writing:

cyber_joseph = cyber_joe

Now the object referenced by the name cyber_joe also has the name cyber_joseph.

Assigning a Name: many old names

Now say that we want to give cyber_joe a nickname: cyber_snake. Because cyber_joe has two names so far, we can use either of the following lines to give him his nickname:

cyber_snake = cyber_joe
cyber_snake = cyber_joseph

Reusing a Name

Suppose now cyber_joe has a kid: baby_snake. cyber_joe desides that he want's to give his name to his newborn child. We can do this easily with:

cyber_joe = baby_snake

But now, because a name can only reference one object, cyber_joe no longer references the snake we used to call cyber_joe. Instead cyber_joe now references baby_snake. This does not mean that we have lost the old cyber_joe because his object is still referenced by the names cyber_joseph and cyber_snake. So long as he as at least one name, we won't have lost him.

Some Rules

Rules for Creating Names

  • Names can only be composed of letters, numbers, and underscores
  • Names cannot start with a number
  • Names are case sensitive: cyber_joe is different from Cyber_joe
  • The words in the following table should not be used as names
and as assert break class continue def
del elif else except exec False finally
for from global if import in is
lambda None not or pass print raise
return True try while with yield
