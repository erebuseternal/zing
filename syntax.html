Syntax for Tools Written in Multiple Lines

In what follows we are going to talk about tools in python that always require many lines of code. Each of these tools allow us to run code in a different way from the rest a program. And each of these tools are made from blocks that share the same written form. I would like to show you that form now.

Each blocks takes:

  1. a keyword to specify what kind of block this is
  2. some identifying text to differentiate this block from others like it
  3. a colon
  4. and, in the following lines, the code attached to this block

Now the rules for the keyword, identifying text, and code change depending on the block type. We will learn these later. But there is one rule that holds true for all block types:

the code must always be indented right with respect to the keyword.

This is because the code in your block is going to get run differently from the code outside it. Therefore you indent the code inside so python can distinguish it from the rest of your code.
