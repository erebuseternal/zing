Objects

Python Objects have exactly two components. They have attributes and they can take actions.

Our Example

To show you how python objects work, I will compare a physical object with a python object. Our objects will be joe the pet snake and his cyber twin cyber_joe. First I will show you how to get their attributes, and then I will show you how to make an action take place.

Getting Attributes

Some of the attributes snakes have are:

  1. color
  2. fang length
  3. is the snake hungry?

How do we determine joe and cyber_joe's attributes?

Physical World

We would determine joe's attributes by:

  1. looking at joe
  2. measuring joe's fangs
  3. asking joe's owner when last joe was fed

Coding World

Getting attributes in the coding world is far simpler. cyber_joe's attributes can be accessed with these lines of code:

  1. cyber_joe.color
  2. cyber_joe.fang_length
  3. cyber_joe.is_hungry

How to Get an Attribute

To get an attribute from an object you write:

object.attribute

  1. object's name
  2. period
  3. the attribute's name

Taking Actions (Calling Methods)

How do we make each of our snakes eat two mice?

Physical World

We give joe the mice and he gobbles them up. Mission accomplished.

Coding World

In contrast, objects in the coding world don't do anything until we tell them to. Therefore we have to tell cyber_joe to eat.

In coding lingo we would say that we need to call cyber_joe's eat method. This is because, in coding, the actions an object can take are often called methods. So, how do we call cyber_joe's eat method? With this line of code:

cyber_joe.eat(mouse1, mouse2)

This method will perform whatever actions constitute eating in cyber-joe's world. Part of that will be cyber_joe.is_hungry will now be false.

How to Call a Method

To call a method on an object you write:

object.method(input_object1, input_object2, ...)

  1. object's name
  2. period
  3. method's name
  4. in parenthesis the other objects in the action separated by commas
